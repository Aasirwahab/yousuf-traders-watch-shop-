Building **ShopLedger**, a sales/expense/profit tracker ("digital shop notebook") for **ANY** small local business that can't afford a POS system — clothing, cosmetics, hardware, electronics, grocery, pharmacy, gift shops, etc. It is **general-purpose, NOT watch-specific**; a watch shop is only the first pilot/example. (User corrected me on this — don't pigeonhole it as "watch shop".) App lives in the `shopledger/` subfolder of the working dir; the full spec is `shopledger_full_product_design_and_architecture.md` (its **Revision 1.1** section holds the locked decisions and overrides the rest). GitHub: https://github.com/Aasirwahab/expensive-tracker (branch `main`).

**Stack (versions verified at build time):** Next.js 16 (App Router, Turbopack, `proxy.ts` — NOT middleware.ts), React 19, pnpm, Prisma 7 (driver adapter `@prisma/adapter-pg` + `pg`; runtime uses pooled `DATABASE_URL`, migrations use `DIRECT_URL` via `prisma.config.ts`; generated client at `src/generated/prisma/client`), Neon Postgres (ap-southeast-1), Clerk Core 3 / `@clerk/nextjs` v7 (`<Show when="signed-in">` replaced SignedIn/SignedOut; `auth()` is async), Tailwind v4 (CSS `@theme` in globals.css), Recharts 3, lucide-react 1.x.

**Locked decisions:** money = whole rupees as Prisma `Int` (no cents, no BigInt); quantities `Int`; `stock_movements` ledger is the source of truth and `Product.stockQuantity` is a cache; the sale's stock update must be a conditional UPDATE (no read-then-decrement) to avoid the oversell race; weighted-average cost deferred. Build the **trimmed core** first.

**Access model (confirmed):** two roles. Owner = sees everything (financial dashboard, profit, expenses, reports, team) and invites staff; first user to create the business becomes Owner. Staff/employee = can record sales + view products/stock + their own recent entries only — NO profit/cost/expenses/reports/exports. The financial dashboard is owner-only; staff get a limited Quick-Sale view. Enforce on the server, not just the UI.
